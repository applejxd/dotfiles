# プロダクトコンテキスト

## 解決する問題

開発者は複数のマシンやプラットフォームで作業することが多く、以下の課題に直面しています：

1. 各マシンで同じ開発環境を手動で再構築する時間的コスト
2. 設定の不一致によるワークフローの混乱
3. 設定変更の追跡と同期の難しさ
4. 新しいマシンのセットアップ時の設定忘れ

このdotfilesプロジェクトは、これらの問題を解決するための一元管理されたリポジトリを提供します。

## 想定ユーザー

- 複数のマシンで作業する開発者
- 一貫した開発環境を求めるチームメンバー
- 環境構築を自動化したいシステム管理者
- 設定を定期的にバックアップしたいユーザー

## 使用シナリオ

### 新しいマシンのセットアップ

1. リポジトリをクローン
2. `deploy.sh`または`install.sh`を実行
3. プラットフォーム固有のセットアップスクリプトが自動的に実行される
4. 必要なソフトウェアがインストールされ、設定ファイルがシンボリックリンクで配置される

### 設定の更新

1. ローカルで設定ファイルを編集
2. 変更をリポジトリにコミット
3. 他のマシンでプルして変更を同期
4. 必要に応じて`deploy.sh`を再実行してシンボリックリンクを更新

### カスタマイズ

1. プラットフォーム固有のディレクトリで設定を調整
2. 個人用の設定は`personal.sh`で管理
3. 必要に応じてインストールスクリプトを修正

## 成功指標

- セットアップ時間の短縮（手動vs自動）
- 設定の一貫性の向上
- 環境間の切り替えによる生産性の向上
- トラブルシューティング時間の削減
