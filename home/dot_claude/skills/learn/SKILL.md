## 手順

### 1. 現状の把握

- プロジェクトルートの `AGENTS.md`, `CLAUDE.md`, `.claude/CLAUDE.md` を探し、読み込む
- 存在しない場合、その旨を最終レポートに含める

### 2. 知見の抽出

これまでの会話を振り返り、以下の観点で追記すべき情報を特定する:

- **ビルド・テスト手順**: 新たに判明したコマンドや手順
- **コードスタイルの暗黙知**: 合意した命名規則、設計方針、パターン
- **アーキテクチャ上の決定**: 技術選定、依存関係、ディレクトリ構成の変更
- **落とし穴・注意事項**: デバッグで判明したハマりポイント、既知の制限
- **ワークフローの変更**: Git 運用、デプロイ手順、CI/CD の更新
- **ユーザーの好み**: コーディングスタイルや作業フローの嗜好

### 3. 肥大化防止チェック

差分を作成する前に以下を必ず確認する:

- **重複排除**: 既存の記載と意味的に重複する内容は追加せず、既存項目の更新で対応
- **統合整理**: 同一セクションに類似の項目が 5 つ以上並ぶ場合は上位概念でまとめ直す
- **詳細の分離**: 3 行を超える詳細は AGENTS.md 本体に書かず、別ファイル（`.claude/rules/`, `docs/` 等）に切り出して参照リンクを貼る
- **陳腐化チェック**: 新たな知見で古くなった記載があれば更新・削除を提案

### 4. AGENTS.md に直接反映

- 肥大化防止チェックを通過した変更を AGENTS.md に直接書き込む
- 詳細の分離が必要な場合は `.claude/rules/` や `docs/` にファイルを作成し、AGENTS.md には参照リンクのみ記載

### 5. 変更サマリー

親エージェントに返すサマリーは以下の形式で**簡潔に**まとめる:

```
## AGENTS.md 更新完了

変更前: <N> 行 → 変更後: <M> 行

### 変更内容
- [セクション名]: <変更の要約>（理由: この会話で○○が判明したため）

### 外部ファイル作成
- `<ファイルパス>`: <内容の要約>

ユーザーは `git diff` で変更内容を確認し、不要な場合は `git checkout` で戻せます。
```

## 制約

- **会話に根拠のない情報を追加しない**: 推測で記載しない
- **変更理由を明記する**: 「この会話で○○が判明したため」等
- **AGENTS.md は簡潔に保つ**: 詳細は別ファイルに分離し参照で繋ぐ
- **言語**: ユーザーの使用言語に合わせる
