# AGENTS.md オンボーディング

既存または新規プロジェクトに対して、対話的に **AGENTS.md**（Claude Code 向けには **CLAUDE.md**）を生成する。

## ステップ 1 — プロジェクト調査

まずプロジェクトの現状を把握する。

1. **既存 AGENTS.md / CLAUDE.md の有無を確認**  
   - ルートおよびサブディレクトリに `AGENTS.md`, `CLAUDE.md`, `.claude/CLAUDE.md` が存在するか調べる
   - 存在する場合は内容を読み取り、更新モードで進める
2. **プロジェクト構造の自動解析**  
   - ディレクトリ構造、主要ファイル、設定ファイル（`package.json`, `pyproject.toml`, `Cargo.toml`, `go.mod` 等）を確認
   - 使用言語・フレームワーク・パッケージマネージャーを特定
3. **既存ドキュメントの読み取り**  
   - `README.md`, `CONTRIBUTING.md`, `.editorconfig`, リンター設定等があれば参照

## ステップ 2 — ユーザーへのヒアリング

ステップ 1 の調査結果をユーザーに共有したうえで、以下を **対話的に** 確認する。不明点はスキップ可とし、最低限必要な項目だけで進められるようにする。

### 必須ヒアリング項目
- [ ] **ビルド・テストコマンド**: プロジェクトのビルド方法、テスト実行方法
- [ ] **コードスタイル規約**: インデント、命名規則、言語固有のルール

### 任意ヒアリング項目
- [ ] **プロジェクト概要の補足**: 自動解析で把握しきれない目的・背景
- [ ] **アーキテクチャ上の制約**: 設計方針、使用禁止ライブラリ等
- [ ] **Git ワークフロー**: ブランチ戦略、コミットメッセージの規約
- [ ] **デプロイ手順**: ステージング・本番リリースのフロー
- [ ] **セキュリティ上の注意事項**: 機密ファイル、環境変数の扱い
- [ ] **その他の指示**: エージェントに常に守らせたいルール

## ステップ 3 — AGENTS.md の生成

収集した情報を基に AGENTS.md を生成する。以下の原則に従うこと。

### 記述原則

1. **具体的であること** — 曖昧な指示を避け、実行可能な指示を書く
   - ✅ `npm test -- --watch=false` を実行してからコミット
   - ❌ 変更をテストすること
2. **簡潔であること** — 冗長な説明は避け、要点のみ記載
3. **Progressive Disclosure** — ルートの AGENTS.md は概要に留め、詳細は別ファイル（`docs/`, `.claude/rules/` 等）への参照で対応

### 出力テンプレート

```markdown
# Project Name

<1-2行のプロジェクト概要>

## Tech Stack

- Language: ...
- Framework: ...
- Package Manager: ...

## Build & Test

- Build: `<command>`
- Test: `<command>`
- Lint: `<command>`

## Project Structure

<主要ディレクトリの役割を箇条書き>

## Code Style

<インデント、命名規則など>

## Architecture

<重要な設計方針・制約>

## Workflow

<ブランチ戦略、コミット規約など>

## Additional Rules

<その他エージェントが守るべきルール>
```

### 出力先の選択

ユーザーに以下のどちらに出力するか確認する:

- `AGENTS.md`（ツール横断で利用可能）
- `CLAUDE.md`（Claude Code 専用）
- 両方（共通部分を AGENTS.md、Claude 固有を CLAUDE.md）

## ステップ 4 — レビューと確定

1. 生成した AGENTS.md のドラフトをユーザーに提示
2. フィードバックを受けて修正
3. 承認後にファイルとして書き出す

## ポリシー

- **禁止**: ユーザーの承認なしにファイルを書き出すこと
- **要求**: 各ステップでユーザーの確認を取りながら進めること
- **言語**: ユーザーの使用言語に合わせる（日本語で話しかけられたら日本語で応答）
