# レビュー・チェックリスト（AGENTS.md）

## A. READMEとの差別化

- README向け情報（製品説明・導入手順）が過剰に混入していない
- AGENTS向け情報（運用ルール・安全境界・検証）が中心

## B. 実行可能性

- install/build/test/lint/typecheck が実在コマンドである
- CIの実行コマンドと大きく矛盾しない
- コマンドに必要な引数が具体的に書かれている

## C. 安全性

- Always / Ask first / Never が定義されている
- secrets / 本番設定 / 破壊的操作の禁止が明示されている
- 依存追加・DB変更・CI変更の承認条件が明示されている

## D. 保守性

- 60〜150行程度に収まり、可読性が高い
- 不明点は `TODO: 要確認` として隔離されている
- ディレクトリ責務が明確で、越境変更ルールがある

## E. レビュー出力品質

- 差分要約がある
- 影響範囲が書かれている
- 次のアクション（承認/修正）が明確
